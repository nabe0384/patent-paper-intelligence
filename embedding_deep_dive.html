基本概念

Embeddingとは何か

テキスト（文章・単語）を数値ベクトルに変換する技術。コンピューターは文字列を直接比較できないが、ベクトルなら数学的に「近さ」を計算できる。

「空調の省エネ制御」と「HVACエネルギー削減」はキーワードが違うのに意味が同じ→ ベクトルが近い。これがEmbeddingの核心。

採用理由

従来手法との比較

なぜキーワード検索ではなくEmbeddingを使うのか。3つの観点で比較する。

観点キーワード検索（TF-IDF）Embedding（意味検索）
同義語❌ 「HVAC」と「空調」は別物扱い✅ 同じ意味として近くに配置
文脈理解❌ 単語の出現頻度のみ✅ 文全体の意味を把握
多言語❌ 言語ごとに別処理が必要✅ 日英混在でも対応
実装コスト✅ シンプル◯ モデル読込が必要
精度△ キーワード一致依存◎ 意味で判断

論文・特許データに特に有効な理由

① 同じ技術を異なる表現で書く著者が多い
例：「異常検知」「故障検出」「アノマリー検出」→ 全部同じ意味
② 日本語・英語が混在している論文が多い
例：タイトルは日本語、本文に英語の技術用語が混在
③ 図表・数式が多く、テキスト量自体が少ない
→ キーワードの出現頻度に頼るTF-IDFは不利
④ 「この論文と似た研究」を意味レベルで探したい
→ 意味ベースのEmbeddingが最適
仕組み

TransformerベースのEmbeddingモデルの仕組み

multilingual-e5はBERTベースのTransformerモデル。テキストを以下の3ステップで処理する。

正規化（normalize_embeddings=True）の意味

ベクトルの長さを1に正規化することで、文章の長さに関わらず公平な比較ができる。長い文章は数値が大きくなりがちだが、正規化で方向だけで比較できるようになる。

import numpy as np vec = np.array([0.82, -0.14, 0.56, 0.71]) norm = np.linalg.norm(vec) # ベクトルの長さ = 1.18 normalized = vec / norm # 長さ1に正規化 # → [0.69, -0.12, 0.47, 0.60] # 正規化後はコサイン類似度 = 内積だけで計算可能（高速） similarity = np.dot(vec_a, vec_b) # 正規化済みなら内積 = cos類似度
数学的根拠

コサイン類似度とは

2つのベクトルの「向きの近さ」を-1〜1のスコアで表す指標。1に近いほど似ている、0は無関係、-1は正反対を意味する。

実際のスコア感覚（今日の結果から）

今日の実験では類似度80〜88%が出た。これは実際にどういう意味か。

80%台は「意味的に強く関連している」水準。60%以下は「無関係」の目安。この差でカテゴリを判定している。

from sklearn.metrics.pairwise import cosine_similarity import numpy as np # 2つの文をEmbedding化してスコアを計算する例 vec_a = model.encode(["passage: 空調システムの省エネ制御"]) vec_b = model.encode(["passage: HVACのエネルギー効率化"]) vec_c = model.encode(["passage: テナント満足度の調査方法"]) print(cosine_similarity(vec_a, vec_b)[0][0]) # → 0.91（近い） print(cosine_similarity(vec_a, vec_c)[0][0]) # → 0.62（遠い）
選定根拠

なぜmultilingual-e5-smallを選んだか

今回のPOCで採用した理由は4つある。

① 完全無料・ローカル動作
APIキー不要。インターネット接続なしでも動く。POC段階のコストゼロ運用に最適。
② 日本語と英語の両方に対応
NTTファシリティーズの論文は日本語タイトル＋英語本文が多い。multilingual-e5は100言語以上を1モデルでカバー。
③ 「small」は速度と精度のバランスが良い
384次元・33Mパラメータ。PCのCPUでも数秒で100件処理できる。largeは精度高いが遅い。
④ passage:/query:プレフィックスで精度向上
文書検索に特化した学習設計。論文・特許の検索用途に最適化されている。

モデル比較表（本番移行時の参考）

モデル次元数コスト日本語精度推奨用途
multilingual-e5-small384無料◯POC・社内利用
multilingual-e5-large1024無料◎精度重視の本番
text-embedding-3-small1536有料（安）◎コスパ重視の本番

POC → e5-small、本番 → e5-large or text-embedding-3-smallが現実的な移行パス。コードは1行変えるだけで切り替えられる設計にしてある。
