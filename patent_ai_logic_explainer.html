前処理

スライディングウィンドウでチャンク分割

論文は長文（数千〜数万文字）のため、文書まるごとベクトル化すると情報が平均化されて精度が落ちる。300文字単位・100文字オーバーラップで分割することで、局所的な内容を保ちつつ文脈の連続性も確保する。

chunk_size = 300 # ウィンドウサイズ stride = 100 # オーバーラップ while start < len(text): chunk = text[start : start + chunk_size] chunks.append(chunk) start += chunk_size - stride # 100文字重複

オーバーラップを入れる理由

チャンクの境界で重要な文章が分断されるリスクを防ぐ。100文字のオーバーラップにより、前後のチャンクに同じ内容が含まれるため、文脈が途切れない。

コア技術

Embedding（テキスト → ベクトル）

テキストを384次元の数値ベクトルに変換する。意味が似ている文章はベクトル空間上で近い位置に配置される。今回はmultilingual-e5-smallを使用（日本語・英語の両方に対応した多言語モデル）。

model = SentenceTransformer("intfloat/multilingual-e5-small") emb = model.encode("passage: " + text, normalize_embeddings=True) # → shape: (384,) 384次元ベクトル

なぜ「passage: 」をつけるか

multilingual-e5は検索用途に特化した設計で、文書側には passage:、クエリ側には query: のプレフィックスをつけることで精度が大幅に向上する（モデルの学習仕様）。

HDBSCANクラスタリング

ベクトル化されたチャンクを、意味的な近さでグループに分ける。HDBSCANはk-meansと違い「クラスタ数を事前指定不要」で、密度の低い点を「ノイズ」として扱える。論文・特許のように多様なトピックが混在するデータに向いている。

clusterer = hdbscan.HDBSCAN( min_cluster_size=3, # 最小3チャンクでグループ形成 min_samples=1, metric="euclidean", ) labels = clusterer.fit_predict(embeddings) # -1 = ノイズ（どのクラスタにも属さない）

Claude APIでタグ・キーワード自動生成

クラスタリングで作られた各グループの代表チャンクをClaudeに渡し、「このグループを表す技術領域名とキーワード」を生成させる。クラスタリング → LLMタグ生成の順番が重要（逆にすると精度が落ちる）。

prompt = f""" 以下のテキスト群のトピックをJSONで返してください。 テキスト: {sample_texts} 出力形式: {{"topic_name": "トピック名", "keywords": ["kw1", "kw2", "kw3"]}} """ response = claude.messages.create( model="claude-haiku-4-5-20251001", messages=[{"role": "user", "content": prompt}] )

APIキーなし時のフォールバック

Claude APIが使えない場合は、チャンク内の頻出単語をキーワードとして代替使用する。精度は落ちるが動作自体は継続する設計になっている。

マッピング

事前定義カテゴリへのマッピング

検出したトピックを、あらかじめ定義した仮説カテゴリ（A〜G）に自動分類する。手法はシンプルで、トピックのベクトルと各カテゴリ説明文のベクトルのコサイン類似度を計算し、最も近いカテゴリに割り当てる。

topic_emb = model.encode(["passage: " + topic_text]) cat_embs = model.encode(["passage: " + cat for cat in categories]) # コサイン類似度で最近傍カテゴリを特定 sims = cosine_similarity(topic_emb, cat_embs) best_category = categories[argmax(sims)] score = max(sims) # 類似度スコア（80〜90%台が目安）
